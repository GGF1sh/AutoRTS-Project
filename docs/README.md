# docs — 開発ドキュメント

このフォルダは「**誰が・いつ・どのバージョンに対して・何を言い/作ったか**」を後から辿れるようにするための記録置き場です。

## フォルダ構成

```
docs/
├── README.md            ← このファイル（見方とルール）
├── VERSION_HISTORY.md   ← バージョン ⇄ フェーズ ⇄ コミットの対応表
├── phases/              ← 各フェーズを「どう作ったか」の実装記録
│   └── Phase1.md
└── ai-log/              ← 4AIごとの発言ログ（時系列）
    ├── ChatGPT.md
    ├── Claude.md
    ├── Gemini.md
    └── Grok.md
```

## バージョン規約

- バージョンは **vメジャー.マイナー.パッチ** 形式。
- **マイナー番号 = フェーズ番号**に対応させる。
  - 例: `v0.1.x` = Phase 1、`v0.6.x` = Phase 6。
- フェーズ内の小さな修正はパッチ番号を上げる（`v0.1.0` → `v0.1.1`）。
- どのバージョンがどのコミットかは [VERSION_HISTORY.md](VERSION_HISTORY.md) で対応づける。

## 4AIの役割（合意済み）

| AI | 役割 |
|---|---|
| ChatGPT | 企画整理・仕様書・タスク分解 |
| Claude（実装担当） | コード実装・修正・リファクタリング |
| Gemini | Godot公式ドキュメント調査・実装方式の比較 |
| Grok | 企画の批判レビュー・別案出し・スコープ膨張のチェック |

## 更新ルール

- **フェーズを実装したら** `phases/PhaseN.md` を1枚書く。
- **AIから意見をもらったら** その AI の `ai-log/<AI名>.md` に1行追記する（日付・対象Ver・意見・採否）。
- **動いてコミットしたら** `VERSION_HISTORY.md` に1行追記する。
- 迷ったら「未来の自分が読んで分かるか？」を基準にする。
