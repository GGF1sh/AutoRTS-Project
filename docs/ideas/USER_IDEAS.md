## 1. 戦闘ログ → AIで小説化

**やりたいこと**: 保存した戦闘ログ＋敵味方のキャラ設定を、Gemini / ChatGPT / Grok / Claude などに渡すと「こんな感じに戦闘したのでは」という小説・情景描写を出力してくれる。

**現状の土台**: v0.8.0 で「戦闘ログを保存」機能を実装済み。`user://logs/battle_*.txt` に〈参加ユニットの設定＋戦闘の流れ〉が出る。これをそのままAIに貼れる。
（保存先の実体パスは保存時のログに表示。例: `C:\Users\<名前>\AppData\Roaming\Godot\app_userdata\gunbitrts\logs\`）

**プロンプト雛形（叩き台）**:
```
あなたは戦記小説の作家です。以下は2Dトップダウン分隊バトルの戦闘ログと、
登場ユニットの設定です。これを元に、各キャラの性格を踏まえた短い戦闘シーンを
小説形式で書いてください。誇張しすぎず、ログの出来事の順序は守ってください。

# 登場ユニット
（ここに保存ファイルの「参加ユニット」セクションを貼る）

# 戦闘の流れ
（ここに保存ファイルの「戦闘の流れ」セクションを貼る）
```

**将来の自動化案**: ログ保存時に、上記プロンプト込みの `.txt` を一緒に出力する。さらに先で API 連携も検討（ただしコスト/制限に注意）。

**やりたいこと**: 各キャラに好きな画像を割り当て、性格などの設定を持たせる。

**実装メモ**:

- 画像は Godot の `res://assets/portraits/` に置き、Sprite2D かUIに表示。

## 3. ガンビット連動セリフ（状況別ボイス/テキスト）

**やりたいこと**: HP減少・強スキル使用・MP減少・大ダメージ・後退 等の状況でキャラがセリフを喋る。ガンビット（条件→行動）に紐づける。

**仕様アイデア**:
- セリフは状況キーごとに複数候補を持ち、ランダムで1つ選ぶ。
- **発生確率**を % で設定可能（毎回喋るとうるさいので、例: 後退時30%）。
- **文字色**もセリフごとに設定可能。
- データ例（将来の `data/dialogue.json` 案）:
```json
{
  "ミナ(衛生)": {
    "on_retreat":  { "chance": 40, "color": "#88ccff",
                     "lines": ["下がる！", " here is まずい…"] },
    "on_heal":     { "chance": 60, "color": "#66f28c",
                     "lines": ["治療します！", "しっかりして！"] },
    "on_low_hp":   { "chance": 80, "color": "#ff8888",
                     "lines": ["もう…ダメかも…"] }
  }
}
```
- トリガー候補: `on_low_hp` / `on_big_damage` / `on_retreat` / `on_heal` / `on_skill` / `on_mp_low` / `on_kill` / `on_death`。
- 実装は Unit の `_set_action` / `take_damage` / `_die` などの既存フックから「状況キー」を発火させ、確率判定して戦闘ログ（または吹き出し）に色付きで出す。

## 5. ComfyUI 画像/3D生成（別AI担当）

**環境**: ユーザーは ComfyUI + NVIDIA RTX 4070 Super 所有。画像生成可、やり方次第で3D生成も可能。

**運用方針**: ComfyUI 用のプロンプト・ワークフロー・モデル/ノード提案は **別AI（Grok/Gemini等）に投げる**（Claudeは生成系プロンプトの制限が厳しめのため）。

**用途**: キャラのポートレート、ユニットアイコン、背景タイル、UIアイコン、（将来）3Dモデル。一貫性のため LoRA/ControlNet/IP-Adapter 等の活用を検討。
