## 目的

「仲間が条件に応じて賢く動く」核心＝**ガンビットAI**を入れる。条件→行動のリストを上から評価し、最初に成立した1件だけを実行する。これでロールごとに動きが変わる。

## 変更したファイル

| ファイル | 変更内容 |
|---|---|
| `Unit.gd` | ガンビット評価エンジンを追加。条件判定・行動実行・探索ヘルパー・回復処理・ロール別既定ガンビット。 |
| `Main.gd` | ミナ(Medic)に `heal_power:16` を付与、回復が届くよう `attack_range` を 60→90 に。 |

## ガンビットの仕組み

各ユニットは `gambits = [ {cond, action}, ... ]` を持つ。`_run_gambits()` が毎フレーム上から評価し、最初に成立した行動だけ実行して `return`（FF12方式）。

### 実装した条件（MVP版）

| 条件 | 意味 |
|---|---|
| `self_hp_below` (param) | 自分のHP割合が param 未満 |
| `ally_hp_below` (param) | HP割合 param 未満の味方がいる |
| `enemy_in_range` | 最寄り敵が攻撃射程内 |
| `enemy_too_close` (param) | 最寄り敵が param 未満の距離（近すぎ） |
| `nearest_enemy_exists` | 生きている敵がいる |

### 実装した行動（MVP版）

| 行動 | 意味 |
|---|---|
| `attack_nearest` | 最寄り敵を攻撃（射程外なら接近） |
| `attack_weakest` | HP最小の敵を攻撃（Support用） |
| `retreat` | 最寄り敵から離れる（画面内にクランプ） |
| `move_to_nearest_enemy` | 最寄り敵へ接近 |
| `heal_lowest_hp_ally` | 最も弱った味方を回復（射程外なら接近） |

## ロール別の既定ガンビット

| ロール | 振る舞い |
|---|---|
| Frontline / Raider / Brute | 射程内→攻撃、いなければ接近 |
| Shooter | 自分HP<30%→後退、敵が120px未満に近づく→後退、射程内→攻撃、else接近（カイト） |
| Medic | 自分HP<25%→後退、味方HP<70%→回復、敵射程内→攻撃、else接近 |
| Support | 自分HP<30%→後退、射程内→**最も弱った敵**を攻撃、else接近 |

## ログで「判断」を見せる

行動が切り替わった瞬間だけログを出す（毎フレーム出すとスパムになるため）:
- 「← レン は危険を察知して後退する」
- 「ミナ は負傷した味方を助けに向かう」
- 「✚ ミナ が タロウ を 16 回復」

## 既知の制約 / 次への申し送り

- ガンビットはまだコード内（ロール既定）。**外部編集UIや JSON 化は Phase 8 / Phase 10**。
- `setup()` の `data["gambits"]` に配列を渡せば個別上書きも可能（拡張の足場は用意済み）。
- 回復・攻撃は同じ `attack_cooldown` を共有している。
