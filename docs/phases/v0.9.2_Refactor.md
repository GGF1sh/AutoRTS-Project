# v0.9.2 — Main.gd の分割リファクタ（挙動不変）

- **日付**: 2026-06-15 / **担当**: Claude / **状態**: 実装完了（実機確認待ち）

## 背景（ユーザー指摘）

Main.gd が肥大化し、AIが編集のたびに全体を読む＝**トークンの無駄**。特に `_fallback_allies()` のキャラ生データが `data/units.json` と重複してMainにも書かれていた。**ファイル分割を最優先**。

## 方針

**Unit.gd のインターフェース（`battle.*` 呼び出し）は一切変えない**純粋なリファクタ。Main の中身を役割ごとに別ファイルへ切り出し、Main は司令塔（接続役）だけにする。

## 分割後の構成

| ファイル | 役割 | 行数の目安 |
|---|---|---|
| `Main.gd`（class_name Main）| 司令塔：戦闘管理・配置・勝敗・一時停止・生存キャッシュ・各部品の接続・設定保持 | 〜290 |
| `Hud.gd`（class_name Hud, Node）| UI一式の構築・所有＋ログ蓄積/フィルタ/色分け/保存 | 〜420 |
| `Fx.gd`（class_name Fx, Node2D）| 攻撃/回復エフェクトの描画 | 〜70 |
| `AudioBank.gd`（class_name AudioBank, Node）| 効果音のコード合成・再生 | 〜55 |
| `GameData.gd`（autoload）| JSON読込＋**ユニットのフォールバックデータを吸収**（Mainから生データ撤去） | — |

## 役割分担の要点

- **設定値**（low_hp_threshold / alarm_on / sfx_on / auto_pause / log_save_*）は Main が保持。Unit は従来通り `battle.low_hp_threshold` 等で参照（互換）。
- **UI操作**は Hud が受け、戦闘設定に関わるものは `_main.set_xxx()` / `apply_preset()` 等へ委譲。ログ表示フィルタ・色は Hud 内で完結。
- **エフェクト**は Fx（ユニットの後に追加 → 上に描画）。戦場の枠は Main._draw。
- **音**は AudioBank。鳴らす可否（alarm_on/sfx_on）の判定は Main 側。
- レイアウト定数（MARGIN/PANEL_W/LOG_H）は Main の const。Hud から `Main.MARGIN` 等で参照。

## 効果

- 戦闘ロジックを触る時に UI/エフェクト/音/データの巨大コードを読まずに済む（編集対象ファイルだけ読めばよい）。
- キャラ調整は `data/units.json`、UIは `Hud.gd`、AIは `Unit.gd` / `data/gambits.json`、と編集箇所が明確化。

## 検証

- Godot 未所持のため実機未テスト。クロスファイルの呼び出し（`_main.*` / `_hud.*` / `_fx.*` / `_audio.*` / `battle.*`）の定義一致を全て確認済み。挙動はv0.9.1と同一の想定。
