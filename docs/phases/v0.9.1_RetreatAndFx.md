# v0.9.1 — 後退の挙動修正＋エフェクト改善

- **日付**: 2026-06-15 / **担当**: Claude / **状態**: 実装完了（実機確認待ち）

## 背景（ユーザー指摘）

実機で確認したところ「**HPが減ると逃走モードになるが際限がない**（特にヒーラーが延々と逃げ続ける）」のが不自然。エフェクトも近接/回復は専用の見た目が欲しい。

## 修正内容

### 1. 「際限のない後退」を解消
- 新アクション **`retreat_to_safe(factor)`**: 最寄りの脅威の射程×factor（最低150px）まで下がったら **その場で警戒（停止）**。延々と逃げない。
- 新アクション **`flee_to_healer(factor)`**: 衛生兵のところまで後退して回復を待つ（衛生兵が居なければ `retreat_to_safe` にフォールバック）。
- 新アクション **`heal_self`**: 衛生兵が自分を治療する（瀕死で逃げ続ける問題の解消）。
- **戦線復帰**: 回復してHPがしきい値を上回ると、ガンビット再評価で後退条件が外れ、自動的に攻撃へ戻る。

### 2. ロール別ガンビットの再設計
| ロール | 振る舞い |
|---|---|
| Frontline / Raider / Brute | **退かずHP0まで戦う盾役** |
| Shooter | 近接130px以内→安全圏へカイト、瀕死(30%)→衛生兵へ退避 |
| Medic | 近接110px→安全圏へ→自己回復(60%)→味方回復→復帰 |
| Support | 瀕死→衛生兵へ退避、射程内→弱点集中 |
| Scout | 瀕死→安全圏へ、射程内→弱点集中 |

→ これで「回復したら戦線復帰」「盾役はHP0まで」「ヒーラーは自己回復」「他キャラはヒーラーへ退避」がガンビットで表現される。`data/gambits.json` 編集で自由に変更可。

### 3. エフェクト改善
- 近接攻撃（射程≤90）＝**斜めに走るスラッシュ**（残り時間で位置が動く）。
- 遠距離＝従来のビーム＋着弾。
- 回復＝対象から立ち上る**うにょうにょ波線**（`draw_polyline` の正弦波）。

## 変更ファイル

- `Unit.gd`: 新アクション3種＋ヘルパー（`_find_nearest_healer` / `_max_enemy_range`）、ロール別ガンビット再設計、攻撃FXに slash/beam 種別。
- `Main.gd`: `_draw` にスラッシュ＆回復波線、`add_attack_fx` に kind 引数。
- `data/gambits.json`: 新アクション反映（`factor` キーで安全距離倍率）。

## 設計メモ

- ガンビットの rule は条件用 `param` と後退系の `factor` を**別キー**に分離（同一キー衝突を回避）。
- 安全距離は「最寄りの脅威の射程×factor」かつ最低150px（トリガー距離より広く取りハンチング防止）。
