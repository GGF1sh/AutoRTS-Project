## 目的

「仲間が画面上で自律的に動いて戦う」最小プロトタイプを作り、ゲームの核を体験できるようにする。グラフィック・世界観・成長要素は後回し。

## 作ったファイル

| ファイル | 役割 |
|---|---|
| `Main.tscn` | 起動シーン（`Main.gd` を付けただけの最小シーン） |
| `Main.gd` | 戦闘管理：ユニット生成・一時停止・戦闘ログUI・勝敗判定 |
| `Unit.gd` | 味方/敵共通のユニット1体（索敵→接近→攻撃、HP、描画） |
| `project.godot` | `run/main_scene="res://Main.tscn"` を追記（起動シーン指定） |

## 機能

- 画面左に味方4体（**丸**）、右に敵3体（**四角**）を配置
- 各ユニットは最も近い相手に近づき、射程内で攻撃しHPを削る
- HPが0で倒れて消える（以後は対象にならない）
- **スペースキー**で一時停止 / 再開（中央上に「● PAUSED」表示）
- 画面下に**戦闘ログ**（最新8行）
- 全滅で「勝利！」/「敗北…」を表示

## 主要な設計判断（なぜそうしたか）

1. **一時停止は独自フラグ方式 `is_paused`**
   - `get_tree().paused` ではなく `Main.is_paused` を各 Unit が `_process` 先頭で見て止まる。
   - 理由: Gemini提案。ProcessMode設定ミスでUIまで止まる事故を避けられ、初心者に分かりやすい。停止中もログ・UIは生きる。

2. **ユニットはコードで生成（`Unit.new()`）**
   - `Unit.tscn` を手で組まず、`Main.gd` の `_spawn_units()` で辞書から生成。
   - 理由: シーンを手組みするミス（ノード階層・リソース設定）を避け、ステータスを1か所で見渡せる。

3. **見た目は `_draw()` で直接描画**
   - 味方＝`draw_circle`、敵＝`draw_rect`。上にHPバーと名前。
   - 理由: 画像素材ゼロでも成立。仕様の「丸/四角でよい」に合致。

4. **ロールはステータス・色の差のみ**
   - Frontline/Shooter/Medic/Support は射程・HP・攻撃力・色を変えただけ。行動は全員「最寄り敵へ接近して攻撃」で統一。
   - 理由: 回復・後退などロール別AIはバックログ上 Phase 6〜7。Phase 1の合意範囲（接近して攻撃）に絞った。
